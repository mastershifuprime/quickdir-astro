In the Astro subreddit, every day the community shares genuinely useful resources — tools, migration stories, tutorials, templates, integrations, and more. But within a few days, it's all lost in the scroll.

**Astro Awesome** was built to fix that.

We're an open-source, community-curated directory that keeps the best Astro resources easily discoverable. Instead of losing great content to the timeline, we collect, categorize, and surface it — so developers can find what they need, when they need it.

### What we collect

- 🛠️ **Tools & Integrations** — Plugins, adapters, CMS connectors, developer tools, and npm packages that extend Astro
- 🎨 **Themes & Templates** — Starter kits, landing pages, blogs, portfolios, docs sites, and more
- 📚 **Tutorials & Guides** — Step-by-step learning resources from beginner to advanced
- 📝 **Articles** — In-depth posts about Astro architecture, best practices, and real-world experiences
- 🚀 **Deployment** — Guides and tools for deploying Astro on Cloudflare, Vercel, Netlify, and more
- 🧩 **CMS & Content** — Headless CMS integrations — Sanity, Strapi, Contentful, Keystatic, Decap, and others
- ⚡ **Performance & SEO** — Tips, tools, and techniques for optimizing Astro sites
- 🔄 **Migration Stories** — Real experiences moving from Next.js, WordPress, React, and other frameworks to Astro

### How it works

We actively scrape and curate resources shared across the Astro community — primarily from Reddit's r/astrojs, but also from GitHub, Twitter/X, and direct submissions. Every resource is categorized, tagged, and ranked so you can browse by type or search for exactly what you need.

**Anyone can submit a resource.** If you've built a tool, written a tutorial, or published a theme — we want to feature it. Every submission is reviewed for quality and relevance.

### Why we built this

As Astro developers ourselves, we noticed the same frustration: great resources getting buried. A fantastic theme shared on Monday is invisible by Friday. A tutorial that could save someone hours of work never reaches them.

We believe the Astro ecosystem deserves a living directory — one that grows with the community and keeps the best work visible.

### Open source

Astro Awesome is built with Astro (naturally) and is fully open source. Contributions, suggestions, and feedback are always welcome.
